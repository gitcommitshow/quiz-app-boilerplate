Submission Guidelines

                        
    
                            
  • Questions should be relevant to data engineering topics (SQL, databases, statistics, ETL, data warehousing, etc.)
    • 
                            
  • Ensure questions are clear, well-formatted, and have correct answers
    • 
                            
  • Provide at least 2 hints that progressively help users understand the concept
    • 
                            
  • For objective questions, provide at least 2 options with one correct answer
    • 
                            
  • For subjective questions, provide keywords and expected answer format
    • 
                            
  • You can use Markdown formatting and Mermaid diagrams in question text
    • 
                            
  • After submitting, copy or download the Markdown file and open a pull request that adds content/questions/<your-slug>.md
    • 
                            
  • All submissions will be reviewed before being merged into the question bank
